MASM32 Educational Simulator

Milestone 56: unsigned DIV now runs with divide-by-zero and quotient-overflow diagnostics alongside the existing diagnostic settings.

Choose how much educational diagnostic help the simulator should show while running your program.

  • Region-only: apply mandatory VM address, region, permission, and .CONST checks only; closest to normal MASM-style memory behavior.
  • Section capacity: report accesses that pass VM checks but fall outside the simulator's known section capacity range.
  • Section image: report accesses that stay in section capacity but leave the declared or initialized section image.
  • Declared object bounds: report accesses that leave or cross a declared data object, useful for catching accidental overrun/underrun bugs.
  • Warning modes continue execution; strict modes stop before mutation.
  • Warn reports reads from .DATA? or ? storage that has not been written yet.
  • Off keeps deterministic zero-filled behavior without the teaching warning.
  • Strict stops before consuming uninitialized-origin bytes.
  • Warn reports non-portable use of flags made undefined by earlier instructions.
  • Off uses the simulator’s deterministic preserved flag value silently.
  • Strict stops before the flag-consuming instruction mutates state.
  • Notices explain accepted MASM compatibility directives that are no-ops, metadata-only, or limited.
  • Off hides these notices without changing execution behavior.
